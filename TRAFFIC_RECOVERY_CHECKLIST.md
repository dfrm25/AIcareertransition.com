# Organic Traffic Recovery Checklist

Use this once per week after publishing or refreshing content.

## Search Console

1. Open the domain property for `aicareertransition.com`.
2. Review the last 28 days against the previous 28 days.
3. Record:
   - Total clicks
   - Total impressions
   - Average CTR
   - Average position
   - Top 10 queries
   - Top 10 pages
4. Submit newly updated URLs for inspection after deploy.
5. Check indexing status for:
   - Homepage
   - `blog.html`
   - Latest blog post
   - `career.html`
   - `prompts.html`
   - persona pages

## GA4

Create or review explorations using:

- Landing page + session default channel group
- Page path + `scroll_depth`
- Page path + `article_read`
- Page path + `outbound_click`
- Page path + `form_submit_success`
- `quiz_complete` by `readiness_segment`
- `prompt_copy` by `prompt_category`

Register these custom dimensions if they are not already available:

- `page_type`
- `content_cluster`
- `readiness_segment`
- `prompt_category`
- `cta_location`
- `scroll_percent`
- `form_type`
- `link_domain`

## Weekly Decision Rule

Pick next week's work from the evidence:

- High impressions + low CTR: rewrite title/meta/intro.
- Low impressions + important page: add internal links and supporting content.
- Good engagement + low traffic: promote externally and cross-link.
- High bounce / no scroll: fix intent match and above-the-fold clarity.

## Six-Week Publishing Queue

1. Technical SEO, measurement events, homepage popular guides, and cluster links.
2. Publish “How to show AI skills on a resume without sounding fake.”
3. Expand marketing and analytics role pages; submit updated URLs.
4. Publish “ChatGPT vs Gemini vs Claude for job search tasks.”
5. Expand product, copywriting, and data science role pages; submit updated URLs.
6. Refresh `tools-comparison.html` using the latest model/tool changes.
